# SpecLedger Roadmap

Project roadmap and upcoming features.

## Completed Features

### Version 0.1.0 (Current)

- ✅ Core CLI functionality (new, init, deps, doctor)
- ✅ Interactive TUI for project creation
- ✅ Dependency management with caching
- ✅ Framework integration (Spec Kit, OpenSpec)
- ✅ Vendor dependencies support
- ✅ Tool detection and validation
- ✅ YAML metadata (specledger.yaml)
- ✅ Migration from legacy format

## Feature Request Process

1. Create a GitHub issue with `proposal` tag
2. Discuss with the community
3. Maintainers review and prioritize
4. Approved proposals added to roadmap

## Contribution Priorities

1. **High Priority**: Bug fixes, security issues, documentation
2. **Medium Priority**: Feature requests from community
3. **Low Priority**: Nice-to-have enhancements

## Timeline Estimates

- **Minor releases** (0.x.y): Monthly as needed
- **Major releases** (x.0.0): Quarterly or as feature set warrants
- **Patch releases**: As needed for critical fixes

## Dependencies Roadmap

### Spec Kit

We track Spec Kit's development and integrate new features as they become available.

### OpenSpec

We track OpenSpec's development and integrate new features as they become available.

### Beads

We track Beads issue tracker and integrate new task tracking features.

## Versioning Policy

SpecLedger follows Semantic Versioning (SemVer):
- **Major version (X.0.0)**: Breaking changes
- **Minor version (0.X.0)**: New features, backward compatible
- **Patch version (0.0.X)**: Bug fixes, backward compatible

## Support Schedule

- **Current version**: Full support
- **Previous minor version**: Maintenance only
- **Older versions**: Best effort

## Announcements

- **Releases**: Posted on GitHub Releases
- **Security**: Posted in GitHub Security Advisories
- **Deprecations**: Announced 3 months ahead of removal
