**Key Deliverables**:

#### 1. pkg/cli/spec/collision.go

**Functions**:

- **CheckFeatureCollision(repoRoot, featureNum string) error**: Main collision checker
- **checkLocalFeatures()**: Scans specledger/ directory for existing feature numbers
- **checkLocalBranches()**: Checks local git branches for feature number prefix
- **checkRemoteBranches()**: Checks remote origin branches for feature number prefix
- **GetNextFeatureNum()**: Auto-detects next available feature number
- **ParseFeatureNum()**: Extracts feature number from branch name

**Collision Detection Logic**:

```
1. Check local features (specledger/###-name directories)
   ↓
2. Check local branches (refs/heads/###-name)
   ↓
3. Check remote branches (origin/###-name)
   ↓
4. Return error if collision found, nil if clear
```

**Error Messages**:
- Local feature: "feature number 600 already exists locally: 600-bash-cli-migration"
- Local branch: "feature number 600 already has a local branch: 600-bash-cli-migration"
- Remote branch: "feature number 600 already has a remote branch: origin/600-bash-cli-migration"

#### 2. pkg/cli/commands/spec_create.go

**SpecCreateOutput struct**:
```json
{
  "BRANCH_NAME": "600-bash-cli-migration",
  "FEATURE_DIR": "/path/to/specledger/600-bash-cli-migration",
  "SPEC_FILE": "/path/to/specledger/600-bash-cli-migration/spec.md",
  "FEATURE_NUM": "600"
}
```

**Flags implemented**:
- `--json, -j`: JSON output mode
- `--number`: Feature number (required, e.g., 600)
- `--short-name`: Short name or description (required)

**Command Flow**:
1. Validate required flags (--number, --short-name)
2. Open git repository
3. Check for feature number collisions (local features, branches, remote)
4. Generate branch name with stop-word filtering and truncation
5. Create new git branch from current HEAD
6. Create feature directory (specledger/###-name/)
7. Copy spec template from embedded FS
8. Output results (JSON or text)

**Template Handling**:
- Reads from embedded.TemplatesFS
- Fallback paths: specledger/.specledger/templates/spec-template.md
- Graceful fallback to minimal template if not found

**Usage Examples**:

```bash
# Basic usage
sl spec create --number 600 --short-name "test-feature"

# With JSON output
sl spec create --number 600 --short-name "add OAuth2 authentication" --json

# Long description (auto-truncated)
sl spec create --number 600 --short-name "very long description that will be truncated automatically" --json

# Stop-words filtered automatically
sl spec create --number 600 --short-name "add a new feature for the user authentication"
# Creates branch: 600-new-feature-user-authentication
```

## Technical Decisions

### 1. Collision Detection Strategy
- **Decision**: Check local features, local branches, AND remote branches
- **Rationale**: Prevent all types of collisions before creating feature
- **Implementation**: Sequential checks with clear error messages

### 2. Required Flags
- **Decision**: Make --number and --short-name required
- **Rationale**: Explicit is better than implicit, prevents accidents
- **Implementation**: Validate flags before proceeding

### 3. Template Fallback
- **Decision**: Use embedded template with fallback to minimal template
- **Rationale**: Always create a spec.md, even if template is missing
- **Implementation**: Multiple path attempts, then inline minimal template

### 4. Branch Creation
- **Decision**: Create branch from current HEAD
- **Rationale**: Standard git workflow, allows feature branches from any point
- **Implementation**: go-git/v5 CheckoutOptions with Create: true

### 5. Stop-Word Filtering
- **Decision**: Filter common words automatically
- **Rationale**: Cleaner branch names, preserve meaning
- **Implementation**: 44-word stop-word list in branch.go
