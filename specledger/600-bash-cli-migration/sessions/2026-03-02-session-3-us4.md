**Update Process**:

6. Sort entries alphabetically
7. Reassemble file: header + Active Tech + manual additions
8. Write atomically (temp file + rename)

**Deduplication Logic**:

- Track seen entries in map
- Preserve original case for output
- Sort alphabetically for consistency

## Technical Decisions

### 1. Marker Preservation
- **Decision**: Use HTML comment markers for manual additions
- **Rationale**: Invisible in rendered markdown, clear boundaries
- **Implementation**: `<!-- MANUAL ADDITIONS START -->` / `<!-- MANUAL ADDITIONS END -->`

### 2. Deduplication Strategy
- **Decision**: Case-insensitive deduplication, preserve original case
- **Rationale**: Avoid duplicates while maintaining readability
- **Implementation**: Lowercase comparison, map tracking, sort output

### 3. Atomic Writes
- **Decision**: Write to temp file, then rename
- **Rationale**: Prevent partial writes on crash
- **Implementation**: os.WriteFile(tmp) + os.Rename(tmp, final)

### 4. Agent File Mapping
- **Decision**: Hardcoded map of 17+ agent types
- **Rationale**: Known locations for each agent
- **Implementation**: Map in NewAgentUpdater() factory

### 5. Default Agent
- **Decision**: Default to "claude" if no agent specified
- **Rationale**: Most common agent type
- **Implementation**: Empty string check in command
