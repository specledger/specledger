## Phase 0: Research Summary

See [research.md](./research.md) for detailed bash script analysis. Key findings:

### Script ‚Üí Command Mapping

| Script | Go Command | Key Logic to Migrate |
|--------|------------|---------------------|
| `common.sh` | `pkg/cli/spec/detector.go` | `get_repo_root()`, `get_current_branch()`, `get_feature_paths()` |
| `check-prerequisites.sh` | `sl spec info` | JSON output, doc discovery, `--require-*` flags |
| `create-new-feature.sh` | `sl spec create` | Branch creation, stop-word filtering, 244-byte limit |
| `setup-plan.sh` | `sl spec setup-plan` | Template copy, feature dir creation |
| `update-agent-context.sh` | `sl context update` | Plan parsing, agent file updates, marker preservation |
| `adopt-feature-branch.sh` | Superseded | Replaced by ContextDetector in 598 |

### Dependencies Eliminated

| Dependency | Used By | Go Replacement |
|------------|---------|----------------|
| `jq` | common.sh | `encoding/json` |
| `grep` | check-prerequisites.sh, update-agent-context.sh | `regexp`, `strings` |
| `sed` | update-agent-context.sh | `strings.Replace`, `regexp` |
| `git` CLI | create-new-feature.sh | `go-git/v5` (already in git.go) |

### Existing Go Code to Reuse

- `pkg/cli/git/git.go`: `GetCurrentBranch()`, `BranchExists()`, `CheckoutBranch()`
- `internal/spec/resolver.go`: Spec resolution patterns
- `pkg/cli/prerequisites/checker.go`: Tool checking patterns

## Phase 1: Design

### Data Model

No database changes - file-based operations only.

**Key Entities** (from spec.md):
- **FeatureContext**: Resolved from current branch
  - `RepoRoot`, `Branch`, `FeatureDir`, `SpecFile`, `PlanFile`, `TasksFile`
- **BranchName**: Generated with stop-word filtering
  - `Number`, `ShortName`, `FullName` (truncated to 244 bytes)
- **AgentContext**: Agent-specific file metadata
  - `AgentType`, `FilePath`, `HasActiveTechnologies`, `HasRecentChanges`

### API Contracts

All commands output JSON with `--json` flag:

```json
// sl spec info --json
{
  "FEATURE_DIR": "/path/to/specledger/600-bash-cli-migration",
  "BRANCH": "600-bash-cli-migration",
  "FEATURE_SPEC": "/path/to/specledger/600-bash-cli-migration/spec.md",
  "AVAILABLE_DOCS": ["research.md", "data-model.md"]
}

// sl spec create --json
{
  "BRANCH_NAME": "600-bash-cli-migration",
  "FEATURE_DIR": "/path/to/specledger/600-bash-cli-migration",
  "SPEC_FILE": "/path/to/specledger/600-bash-cli-migration/spec.md",
  "FEATURE_NUM": "600"
}

// sl spec setup-plan --json
{
  "PLAN_FILE": "/path/to/specledger/600-bash-cli-migration/plan.md"
}

// sl context update --json
{
  "UPDATED_FILES": ["/path/to/CLAUDE.md"]
}
```

### Command Flags

| Command | Flags | Purpose |
|---------|-------|---------|
| `sl spec info` | `--json`, `--require-plan`, `--require-tasks`, `--include-tasks`, `--paths-only` | Feature context and validation |
| `sl spec create` | `--json`, `--number`, `--short-name` | Create new feature branch and spec |
| `sl spec setup-plan` | `--json` | Copy plan template |
| `sl context update` | `--json`, `--agent` | Update agent context files |

### Cross-Platform Considerations

1. **Path separators**: Use `filepath.Join()` everywhere
2. **Line endings**: Use `\n` in output (Go handles this)
3. **Shell escaping**: Not needed - no shell invocation
4. **Git operations**: Use go-git/v5 (already cross-platform)
