# Implementation Plan: Silent Session Capture

**Branch**: `598-silent-session-capture` | **Date**: 2026-03-02 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specledger/598-silent-session-capture/spec.md`

## Summary

Fix session capture to silently skip when user has no access token or no project ID, instead of spamming warning logs on every commit. The approach is minimal: reorder the `Capture()` function to check credentials before project ID, and return silently (nil error) for both cases instead of emitting stderr warnings. Only real upload/metadata failures should produce output.

## Implementation Approach

### Change 1: Reorder credentials check before project ID (capture.go)

Move `auth.LoadCredentials()` call from after project ID resolution to immediately after the git commit/success checks. This ensures:
- Fastest possible exit for unauthenticated users
- No wasteful project ID lookups when credentials don't exist
- `GetProjectIDFromRemote()` no longer fails with confusing auth errors

### Change 2: Silent return for no-credentials (capture.go)

Replace `result.Error = fmt.Errorf("not authenticated: run 'sl auth login'")` with a simple `return result` (where `result.Captured=false`, `result.Error=nil`).

### Change 3: Silent return for no-project-ID (capture.go)

Remove the 3-line `fmt.Fprintf(os.Stderr, ...)` warning block and replace with a simple `return result`.

### Change 4: Add test cases (capture_test.go)

Add tests verifying:
- `Capture()` returns `Error=nil, Captured=false` when no credentials
- `Capture()` returns `Error=nil, Captured=false` when no project ID
- Existing behavior unchanged for happy path and error paths
