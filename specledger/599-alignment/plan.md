## Work Breakdown

### Phase 1: Research & Audit

**US1 & US2 - Layer Analysis and Overlap Audit**

1. Analyze existing layer architecture from 598 plan
2. Audit all 15 AI commands in `.claude/commands/`:
   - Classify: pure-orchestration / has-business-logic / duplicates-CLI
   - Identify specific overlap areas
3. Audit existing skills in `pkg/embedded/skills/` and `pkg/embedded/templates/`:
   - Classify: unique-knowledge / duplicates-help / outdated
   - Identify CLI syntax duplication
4. Document findings in `research.md`

**Output**: research.md with audit tables and analysis

### Phase 2: Create Artifacts

**US1 - Layer Responsibilities Document**

1. Define decision criteria for each layer:
   - L0 (Hooks): When to use event-driven automation
   - L1 (CLI): Data operations, CRUD, standalone tooling criteria
   - L2 (AI Commands): Workflow orchestration, AI-specific logic
   - L3 (Skills): Domain knowledge, progressive loading triggers
2. Create decision flowchart (text-based)
3. Document exceptions and edge cases
4. Reference CLI constitution from 598

**US3 - AI Command Template**

1. Define standard sections:
   - Frontmatter (description, handoffs)
   - Purpose
   - When to Use
   - Outline (numbered steps)
   - Behavior Rules
2. Document CLI invocation patterns (`--json` usage)
3. Provide before/after examples from existing commands

**US4 - Skill Template**

1. Define standard sections:
   - Overview
   - When to Load
   - Key Concepts
   - Decision Patterns
   - CLI Reference (link to `--help`, not duplicate syntax)
2. Document domain knowledge focus
3. Provide example using sl-issue-tracking skill

**US5 - Cross-Layer Interactions**

1. Document L2→L1 launcher pattern with examples
2. Document L3 loading trigger mechanism
3. Document L1→L0 convenience patterns (CLI configures hooks)
4. Provide concrete code examples from existing codebase

**Output**: 5 artifact files in `artifacts/` directory

### Phase 3: Validation

1. Review each artifact against acceptance criteria
2. Cross-reference with 598 CLI constitution
3. Validate templates against 3 existing commands/skills each
4. Update checklists with validation results
